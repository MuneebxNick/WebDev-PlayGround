Semantic Tags

Semantic tags in HTML provide meaning to the web content, making it easier for browsers and search engines tounderstand the structure and purpose of the content.

semantic vs non semantic examples

Semantic tags help improve accessibility and SEO by clearly defining the roles of different parts of a webpage.

Non-semantic tags, on the other hand, do not convey any meaning about their content. They are used purely for styling or layout purposes.

Semantic vs Non-Semantic Tags

Semantic Tags Non-Semantic Tags
<header> <div>
<nav> <span>
<main> <section>
<article> <div>
<footer> <footer> (when used without context)

In the table above, semantic tags like <header>, <nav>, and <article> provide clear meaning about their content, while non-semantic tags like <div> and <span> do not convey any specific meaning.

Common Semantic Tags

  • <header>: Represents the header section of a document or section.
  • <nav>: Defines a navigation section for links.
  • <main>: Represents the main content of the document.
  • <article>: Represents a self-contained piece of content that could stand alone.
  • <section>: Defines a thematic grouping of content, typically with a heading.
  • <aside>: Represents content that is tangentially related to the main content.
  • <footer>: Represents the footer section of a document or section.
  • <figure>: Represents self-contained content, often with a caption.
  • <figcaption>: Provides a caption for the content of a <figure>.
  • <time>: Represents a specific time or date.
  • <address>: Represents contact information for the author or owner of a document or section.
  • <mark>: Represents highlighted text.
  • <blockquote>: Represents a section that is quoted from another source.
  • <cite>: Represents the title of a work or a reference to a source.
  • <code>: Represents a fragment of computer code.
  • <pre>: Represents preformatted text, preserving whitespace and line breaks.
  • <details>: Represents a disclosure widget from which the user can obtain additional information or controls.
  • <summary>: Represents a summary or heading for the content of a <details> element.
  • <dialog>: Represents a dialog box or other interactive component.
  • <progress>: Represents the progress of a task.
  • <meter>: Represents a scalar measurement within a known range, or a fractional value.
  • <time>: Represents a specific time or date.
  • <template>: Represents a template that is not rendered when the page loads but can be instantiated later using JavaScript.

Example Usage

<header>
    <h1>Welcome to My Website</h1>
    <nav>
        <ul>
            <li><a href="#home">Home</a></li>
            <li><a href="#about">About</a></li>
            <li><a href="#contact">Contact</a></li>
        </ul>
    </nav>
</header>
<main>
    <article>
        <h2>Article Title</h2>
        <p>This is the content of the article.</p>
    </article>

    <aside>
        <h3>Related Links</h3>
        <ul>
            <li><a href="#link1">Link 1</a></li>
            <li><a href="#link2">Link 2</a></li>
        </ul>
    </aside>
