CSS Specificity & Cascade Algorithm

Understanding how CSS specificity and the cascade work is crucial for effective styling in web development.

What is Specificity?

Specificity is a ranking system that browsers use to determine which CSS rule applies when multiple rules could apply to the same element.

Specificity Hierarchy

  • Inline styles (highest specificity)
  • ID selectors
  • Class selectors, attributes selectors, and pseudo-classes
  • Element selectors and pseudo-elements (lowest specificity)

Calculating Specificity

Specificity is calculated using a four-part value:

  • Inline styles: 1,0,0,0
  • ID selectors: 0,1,0,0
  • Class selectors, attributes selectors, and pseudo-classes: 0,0,1,0
  • Element selectors and pseudo-elements: 0,0,0,1

The Cascade

The cascade is the process by which browsers determine which CSS rules to apply when there are conflicting styles. The order of precedence is:

  1. Inline styles
  2. IDs
  3. Classes, attributes, and pseudo-classes
  4. Elements and pseudo-elements
  5. Importance (e.g., `!important` overrides all)

Example

/* HTML */
<div class="box" id="unique-box">Hello World!</div>
/* CSS */
.box {
    color: blue; /* Specificity: 0,0,1,0 */
}
#unique-box {
    color: red; /* Specificity: 0,1,0,0 */
}
.unique-box {
    color: green; /* Specificity: 0,0,1,0 */
}
<style>
    .box {
        color: blue; /* Specificity: 0,0,1,0 */
    }
    #unique-box {
        color: red; /* Specificity: 0,1,0,0 */
    }
    .unique-box {
        color: green; /* Specificity: 0,0,1,0 */
    }
</style>
